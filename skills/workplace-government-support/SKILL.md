# 정부지원사업 신청 가이드

## 빠른 참조 (Quick Reference)

### 주요 정부지원사업 기관
| 기관 | 대표 사업 | 지원 대상 | 규모 |
|------|-----------|-----------|------|
| 창업진흥원 | 초기창업패키지, 창업도약패키지 | 창업 3-7년 | 최대 1억 |
| 중소벤처기업부 | 스마트공장, 수출바우처 | 중소기업 | 다양 |
| TIPS | TIPS 프로그램 | 기술창업팀 | 최대 5억 |
| 중소기업청 | 기술개발(R&D) | 중소기업 | 다양 |
| 각 지자체 | 지역 창업지원 | 해당 지역 기업 | 다양 |
| 한국산업기술진흥원 | 산학협력 R&D | 기업+대학 | 다양 |

### 평가 핵심 요소 (우선순위)
1. **사업화 가능성**: 실제로 돈이 될 수 있는가?
2. **문제 해결의 혁신성**: 기존 방법 대비 얼마나 나은가?
3. **팀 역량**: 이 팀이 해낼 수 있는가?
4. **시장성**: 시장이 충분히 큰가?
5. **사회적 가치**: 일자리 창출, 사회 문제 해결

## 주요 지원사업별 신청 전략

### 초기창업패키지 (창업 3년 이내)
```
지원 내용: 사업화 자금 최대 1억 원
평가 방식: 서류 → 발표 평가

핵심 평가 기준:
① 창업 아이템의 혁신성 및 사업성
② 창업팀 구성 및 역량
③ 사업화 목표 및 추진 전략

작성 포인트:
- "왜 이 문제인가?" 명확히
- 타겟 고객 구체적으로 (예: "30대 직장인" X → "서울 거주 IT 직장인 중 건강관리 앱 미사용자" O)
- 경쟁사와의 차별화 수치로 표현
- 3개월/6개월/1년 마일스톤 명확히
```

### TIPS (Tech Incubator Program for Startup)
```
지원 내용: R&D 자금 최대 5억 + 후속 투자
조건: 민간 액셀러레이터의 1억 투자 선행

핵심 평가 기준:
① 기술의 혁신성 및 글로벌 경쟁력
② 팀의 기술 역량 (논문, 특허, 경력)
③ 글로벌 시장 진출 가능성

작성 포인트:
- 특허/IP 현황 반드시 기재
- 글로벌 유사 사례 vs 우리 기술 비교
- 기술개발 로드맵 구체화
```

## 신청서 작성 공통 섹션

### 1. 사업 개요
```
[작성 원칙]
첫 단락에서 모든 것을 파악할 수 있게 작성:

"[회사명]은 [타겟 고객]의 [핵심 문제]를 해결하기 위해
[솔루션]을 개발·보급하는 [업종] 스타트업입니다.

현재 [현재 성과: 고객 수, 매출 등]을 달성했으며,
이번 지원을 통해 [구체적 목표]를 달성하고자 합니다."
```

### 2. 문제 정의 및 솔루션
```
[문제]
① {문제 1}: {구체적 수치로 크기 표현}
   예) "국내 중소기업 73%가 디지털 세금계산서 처리에
       월 평균 8시간을 소비하고 있으나 (출처: 국세청, 2023)"

② {문제 2}: {수치화}

[솔루션]
"당사의 [솔루션명]은 [핵심 기술/방법]을 통해:
① {효과 1}: {수치로 표현, 예: 처리 시간 90% 단축}
② {효과 2}: {수치}
③ {효과 3}: {수치}"

[차별점]
기존 대비 우월한 점 3가지를 표로 작성
```

### 3. 사업화 추진 전략
```
단계별 추진 계획 (지원 기간 내):

1단계 (1-3개월): {목표}
  - 세부 과제: {내용}
  - 산출물: {결과물}
  - 성과 지표: {KPI}

2단계 (4-6개월): {목표}
  - 세부 과제: {내용}
  - 산출물: {결과물}
  - 성과 지표: {KPI}

3단계 (7-9개월): {목표}
  ...

최종 목표 (12개월):
  - 매출: {목표}
  - 고객 수: {목표}
  - 고용 창출: {인원} (중요 평가 항목!)
```

### 4. 팀 소개
```
[팀 소개 핵심]
각 멤버의 "이 사업과 관련된" 경력만 강조

대표이사 {이름}:
  - 현 {회사명} 대표 (창업 X년)
  - 전 {회사명} {직책} X년 경력
  - 핵심 역량: {이 사업에 관련된 역량}

CTO {이름}:
  - 관련 기술 경력 X년
  - 개발한 주요 서비스: {서비스명} (사용자 X만 명)
  - 보유 특허: {건수}

어드바이저 {이름} (있을 경우):
  - {소속/직함}, 분야 전문성 X년
```

## 구현 가이드

### 신청서 작성 타임라인
```
D-30: 공고 확인, 자격 요건 검토
D-21: 자료 수집 시작 (통계, 레퍼런스)
D-14: 목차 및 논리 구조 확정
D-7: 초안 완성, 내부 검토
D-3: 수정 완료, 첨부 서류 준비
D-1: 최종 검토 및 제출
```

### 정량적 근거 확보 방법
- **시장 규모**: 통계청, 한국은행, 업계 협회 자료
- **문제 크기**: 직접 설문 조사 (최소 30명)
- **효과 검증**: 파일럿 또는 베타 테스트 결과
- **경쟁사 비교**: 직접 사용 후 비교 테스트

## 고급 패턴

### 평가위원 관점에서 체크
```
평가위원이 검토 시간은 약 15-30분:
- 첫 페이지에서 "이게 되겠다" 확신 주기
- 도표/그래프 적극 활용 (텍스트 최소화)
- 핵심 수치를 굵게/강조

자주 나오는 감점 포인트:
- "~할 예정입니다" (증거 없는 계획)
- 시장 규모 근거 없는 주장
- 팀 역량과 사업 간 연관성 부족
- 경쟁사 분석 없거나 "경쟁사 없음" 주장
```

### 발표 평가 준비
```
7-10분 발표 + 10-15분 질의응답 일반적

필수 답변 준비:
Q: "이미 있는 서비스와 뭐가 다른가요?"
Q: "수익은 언제부터 날 수 있나요?"
Q: "팀에 이 사업을 할 역량이 있나요?"
Q: "이 지원금으로 뭘 할 건가요?"
Q: "다음 투자는 어떻게 받을 계획인가요?"
```
