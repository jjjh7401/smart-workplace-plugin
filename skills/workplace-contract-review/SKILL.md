# 계약서 검토 및 위험 분석 가이드

> **면책 고지**: 이 스킬은 일반적인 계약서 검토 가이드를 제공합니다.
> 법적 효력이 있는 중요 계약은 반드시 변호사의 검토를 받으세요.

## 빠른 참조 (Quick Reference)

### 계약서 유형별 체크 포인트
| 계약 유형 | 핵심 확인 항목 |
|-----------|---------------|
| 용역/서비스 계약 | 업무 범위, 납품 기준, 대금 지급, 하자 책임 |
| 근로계약서 | 근로시간, 임금, 복리후생, 해고 조건 |
| NDA(비밀유지) | 비밀 정보 범위, 유효 기간, 위반 시 제재 |
| 임대차 계약 | 보증금, 월세, 중도 해지, 원상복구 |
| 투자 계약 | 투자 조건, 지분 희석, 우선권, 이사회 구성 |
| 공급/납품 계약 | 납기, 품질 기준, 클레임 처리, 지체상금 |

### 위험 신호 (Red Flags)
- 일방적 계약 변경권 부여
- 과도한 손해배상 조항 (계약금액의 100% 이상)
- 모호한 업무 범위 ("기타 관련 업무")
- 불합리한 지식재산권 귀속 조항
- 재계약 거절권 없는 자동 갱신

## 계약서 유형별 검토 가이드

### 1. 용역/서비스 계약서 체크리스트
```
■ 업무 범위 (Scope of Work)
□ 수행 업무가 구체적으로 명시되어 있는가?
□ "기타 갑이 요청하는 업무" 등 포괄적 조항이 있는가?
   → 있다면 범위 제한 조항 추가 요청
□ 산출물(Deliverable) 기준이 명확한가?

■ 대금 지급
□ 지급 일정이 명확한가? (예: 완료 후 30일 이내)
□ 지연 지급 시 이자/지체 상금 조항이 있는가?
□ 분쟁 시 지급 보류 조항 확인

■ 지식재산권
□ 작업 결과물의 IP 귀속이 명확한가?
□ 배경 기술(Background IP)은 을이 보유 확인
□ 제3자 도구/라이브러리 사용 제한 확인

■ 하자 및 책임
□ 하자 보수 기간이 명시되어 있는가?
□ 손해배상 책임 한도가 적절한가?
□ 불가항력(Force Majeure) 조항 포함 여부

■ 계약 해지
□ 해지 사유와 절차가 명확한가?
□ 해지 시 대금 정산 방법이 명시되어 있는가?
□ 중도 해지 시 기수행 업무 비용 처리
```

### 2. 근로계약서 체크리스트
```
■ 필수 기재 사항 (근로기준법 제17조)
□ 임금: 기본급 + 각 수당 명시
□ 소정근로시간: 시작/종료 시간 명시
□ 휴일: 주 1회 이상 유급휴일
□ 연차: 법정 연차 보장 (1년 미만: 월 1일)
□ 취업 장소 및 업무 내용
□ 취업규칙, 단체협약 적용 여부

■ 주의 조항
□ "포괄임금제" 확인 → 연장수당 미지급 위험
□ 경업금지/전직 제한 기간 (2년 이내만 유효)
□ 수습 기간 중 임금 감액 (최저임금의 90% 이상)
□ 비밀유지 의무 범위 적절성

■ 유리한 조항 확인
□ 퇴직금: 1년 이상 근무 시 법정 퇴직금 보장
□ 육아휴직 관련 조항
□ 해고 예고 (30일 전 통보 또는 30일치 급여)
```

### 3. NDA(비밀유지계약) 체크리스트
```
■ 비밀 정보 정의
□ "비밀 정보"의 범위가 명확한가?
□ 공지된 정보, 독자 개발 정보 제외 조항 포함
□ 정보 공개 의무 시 사전 통보 조항

■ 유효 기간
□ NDA 기간이 합리적인가? (통상 2-5년)
□ 영업비밀은 별도로 무기한 보호 가능

■ 위반 시 제재
□ 위반 시 손해배상 범위가 명확한가?
□ 가처분 신청 등 긴급 구제 수단 포함

■ 예외 조항 (반드시 포함 요청)
□ 법적 의무에 의한 공개 (수사기관 요청 등)
□ 수령자가 이미 알고 있던 정보
□ 제3자로부터 합법적으로 취득한 정보
```

## 위험 조항 수정 예시

### 불공정 조항 수정 방법
```
[위험 조항 예시 1]
원문: "갑은 어떠한 사전 통보 없이도 본 계약을 해지할 수 있다."
위험성: 을에게 일방적으로 불리
수정안: "갑은 30일 전 서면 통보를 통해 계약을 해지할 수 있으며,
        해지 시점까지의 을의 기수행 업무에 대한 대금을 정산한다."

[위험 조항 예시 2]
원문: "을은 계약 이행 중 발생한 모든 손해를 배상할 책임이 있다."
위험성: 무한 책임 위험
수정안: "을의 손해배상 책임은 해당 계약 금액을 한도로 하며,
        을의 고의 또는 중과실로 인한 손해에 한한다."

[위험 조항 예시 3]
원문: "을이 수행하는 업무에서 발생하는 모든 지식재산권은
       갑에게 귀속된다."
위험성: 배경 기술까지 양도 위험
수정안: "본 계약 수행 결과물(Deliverable)에 한정한 지식재산권은
        갑에게 귀속되며, 을이 사전 보유한 배경 기술 및
        독립적으로 개발한 기술은 을에게 귀속된다."
```

## 구현 가이드

### 계약서 검토 5단계 프로세스
1. **전체 훑기**: 이상한 조항 표시만
2. **섹션별 정독**: 각 섹션 핵심 파악
3. **위험 조항 목록화**: 수정 필요 항목 정리
4. **수정안 제안**: 상대방에게 수정 요청
5. **협상 및 서명**: 합의된 내용 확인 후 서명

### 계약서 서명 전 최종 확인
```
□ 계약 당사자 정보 (사업자번호, 주소) 정확한가?
□ 계약 금액 및 통화 명시되어 있는가?
□ 계약 기간 및 갱신 조건 확인
□ 준거법 및 관할 법원 (서울 권장)
□ 서명/날인 방식 (법인: 법인 도장, 개인: 인감)
□ 계약서 사본 보유 여부
```

## 고급 패턴

### 계약 협상 전략
```
협상 가능 항목 (유연한 것):
- 지급 조건, 납기
- 하자 보수 기간
- 책임 한도

협상 필수 항목 (고집해야 할 것):
- IP 귀속 조항
- 해지 시 정산 방법
- 일방적 계약 변경 금지
```

### 전자계약서 활용
```
국내 전자계약 플랫폼:
- DocuSign, 모두싸인, 이폼사인
- 법적 효력: 전자서명법에 의해 인정
- 장점: 분쟁 시 서명 사실 입증 용이
```
