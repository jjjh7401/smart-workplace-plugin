# 제안서 및 견적서 작성 가이드

## 빠른 참조 (Quick Reference)

### 문서 유형 선택
| 상황 | 문서 유형 | 핵심 포인트 |
|------|-----------|-------------|
| 신규 고객 영업 | 제안서 | 문제 해결 능력, 차별점 강조 |
| 가격 협의 | 견적서 | 세부 항목, 유효 기간 명시 |
| 내부 프로젝트 | 기획안 | ROI, 실행 가능성 강조 |
| 입찰 참여 | 입찰 제안서 | 요구 사항 100% 부합 |

### 제안서 성공 3요소
1. **고객 Pain Point 명확화**: "고객이 겪는 문제"를 먼저 기술
2. **Solution Fit**: 우리 솔루션이 어떻게 해결하는지
3. **증거**: 사례, 데이터, 레퍼런스로 신뢰 확보

### 섹션별 작성 가이드

#### 1. 현황 분석 및 문제 인식
```
1. 현황 분석

1.1 {고객사명} 현황
  현재 {고객사}는 {업계/시장 상황}에서 다음과 같은 도전에 직면해 있습니다:

  주요 현안:
  ① {문제 1}: {구체적 설명 및 영향}
  ② {문제 2}: {구체적 설명 및 영향}
  ③ {문제 3}: {구체적 설명 및 영향}

1.2 AS-IS vs TO-BE
  현재(AS-IS): {현재 상태}
  목표(TO-BE): {달성 목표}
  Gap: {개선 필요 영역}
```

#### 2. 제안 솔루션
```
2. 제안 솔루션

2.1 솔루션 개요
  {솔루션명}은 {한 줄 설명}입니다.

  핵심 가치:
  - {가치 1}: {설명}
  - {가치 2}: {설명}
  - {가치 3}: {설명}

2.2 차별화 포인트
  | 구분 | 경쟁사 | 당사 솔루션 |
  |------|--------|------------|
  | {기준 1} | {경쟁사 수준} | {당사 강점} |
  | {기준 2} | {경쟁사 수준} | {당사 강점} |
```

#### 3. 기대 효과
```
3. 기대 효과

3.1 정량적 효과
  ① 비용 절감: {금액 또는 %} 절감 예상
  ② 업무 효율: {지표} X% 향상
  ③ 매출 기여: {금액 또는 %} 증대 기대

3.2 정성적 효과
  - {효과 1}
  - {효과 2}

3.3 투자 회수 예상 (ROI)
  - 투자 비용: {금액}
  - 예상 절감/수익: {금액/년}
  - 투자 회수 기간: {기간}
```

#### 4. 추진 일정
```
4. 추진 일정

단계 | 내용 | 기간 | 산출물
-----|------|------|-------
1단계 | 요건 정의 및 분석 | {기간} | 요건 정의서
2단계 | 설계 | {기간} | 설계 문서
3단계 | 구현/개발 | {기간} | 결과물
4단계 | 테스트 | {기간} | 테스트 결과서
5단계 | 오픈/이관 | {기간} | 완료 보고서
```

## 구현 가이드

### 제안서 작성 프로세스
1. **고객 사전 리서치**: 고객사 홈페이지, 뉴스, 공시 자료 분석
2. **Pain Point 가설 수립**: 예상 문제 3-5개 도출
3. **솔루션 매핑**: 각 Pain Point별 우리 솔루션 연결
4. **차별화 요소 정리**: 경쟁사 대비 우위 3가지
5. **레퍼런스 선정**: 유사 업종/규모 성공 사례
6. **가격 전략**: 단일 가격 vs 옵션 구성

### 견적서 필수 확인 사항
- 금액: 공급가액 + VAT 구분 명확히
- 범위: 포함/미포함 항목 명시
- 유효기간: 반드시 명시 (미명시 시 분쟁 발생)
- 결제 조건: 선금/잔금 비율 및 시점

## 고급 패턴

### 입찰 제안서 체크리스트
- RFP(제안요청서) 요구 사항 100% 반영 확인
- 평가 기준 항목별 배점 숙지
- 제출 형식 (파일 형식, 페이지 수) 준수
- 제출 기한 D-3일 완성 목표

### 가격 협상 여지 남기기
- 견적서에 "옵션" 또는 "별도 협의" 항목 포함
- 기본/표준/프리미엄 3단계 구성 제안
