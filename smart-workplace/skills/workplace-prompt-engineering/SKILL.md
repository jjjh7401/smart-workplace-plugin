# AI 프롬프트 엔지니어링 가이드

## 빠른 참조 (Quick Reference)

### 프롬프트 기본 공식
```
역할 지정 + 맥락 제공 + 명확한 요청 + 형식 지정 + 예시 제공
```

### 핵심 기법 한눈에
| 기법 | 설명 | 언제 사용 |
|------|------|-----------|
| Role Prompting | AI에게 역할 부여 | 전문적 답변 필요 시 |
| Few-shot | 예시 2-3개 제공 | 특정 형식 원할 때 |
| Chain-of-Thought | 단계적 사고 유도 | 복잡한 분석 필요 시 |
| System Prompt | 기본 설정 정의 | 일관된 AI 동작 원할 때 |
| Constraint | 제한 조건 설정 | 범위 제한 필요 시 |

## 업무별 프롬프트 템플릿

### 1. 이메일 작성 프롬프트
```
[기본 템플릿]
당신은 한국 비즈니스 커뮤니케이션 전문가입니다.
다음 조건에 맞는 비즈니스 이메일을 작성해 주세요:

- 상황: {상황 설명}
- 수신자: {이름/직책/회사} ({관계: 고객/상사/동료})
- 목적: {이메일의 목적}
- 핵심 전달 사항: {내용}
- 격식 수준: {격식체/일반체}
- 첨부파일: {있음/없음, 내용}

형식: 제목 + 본문 + 서명 형식으로 작성

[예시 활용]
---
상황: 3월 12일에 진행된 미팅 감사 인사 + 제안서 발송 예정 안내
수신자: ABC회사 구매팀 김부장 (외부 고객)
목적: 감사 인사 및 제안서 발송 일정 안내
핵심 전달 사항: 다음 주 화요일까지 제안서 발송 예정
격식 수준: 격식체
---
```

### 2. 보고서 초안 작성 프롬프트
```
[보고서 작성 프롬프트]
당신은 {업종} 분야 전문 컨설턴트입니다.
다음 데이터를 분석하여 경영진 보고서를 작성해 주세요.

보고서 유형: {유형: 주간/월간/프로젝트 현황}
독자: {임원/팀장/전 직원}
데이터:
{데이터 붙여넣기}

작성 형식:
1. 핵심 요약 (3줄 이내)
2. 주요 지표 현황 (표 형식)
3. 분석 및 인사이트
4. 권고 사항

어조: 간결하고 객관적으로, 두괄식으로 작성
```

### 3. 데이터 분석 프롬프트
```
[데이터 분석 프롬프트]
다음 데이터를 분석하고 비즈니스 인사이트를 도출해 주세요.

분석 목적: {목적}
데이터:
{데이터}

다음 질문에 답해 주세요:
1. 가장 주목할 만한 트렌드는 무엇인가?
2. 이상치 또는 예외 사항이 있는가?
3. 원인으로 가능한 것들은?
4. 개선을 위한 제안 3가지

분석 결과를 비기술적 경영진도 이해할 수 있는 언어로 작성해 주세요.
```

### 4. 계약서/법률 문서 검토 프롬프트
```
[문서 검토 프롬프트]
당신은 한국 기업 계약 전문 자문가입니다.
다음 계약서를 검토하고 위험 요소를 분석해 주세요.

계약서 유형: {유형}
당사 입장: {갑/을}

검토 기준:
1. 위험 조항 (불공정 조항, 과도한 책임 등)
2. 누락된 보호 조항
3. 모호한 표현 (분쟁 발생 가능성)
4. 수정 권고 사항 (구체적으로)

계약서 내용:
{계약서 붙여넣기}

※ 법적 자문이 아닌 일반적인 검토 의견임을 인식하고 답변해 주세요.
```

### 5. 아이디어 브레인스토밍 프롬프트
```
[브레인스토밍 프롬프트]
당신은 창의적인 비즈니스 전략가입니다.

주제: {주제}
맥락: {현재 상황 또는 문제}
제약 조건: {예산, 시간, 인원 등}

다음 방식으로 아이디어를 제시해 주세요:
- 현실적이고 즉시 실행 가능한 아이디어: 5개
- 중장기적 혁신 아이디어: 3개
- 의외의 발상 전환 아이디어: 2개

각 아이디어에 대해:
- 핵심 내용 (2줄)
- 예상 효과
- 선행 조건 또는 주의사항
```

## 고급 프롬프트 기법

### Chain-of-Thought 프롬프트
```
복잡한 문제 해결 시 단계적 사고 유도:

"다음 문제를 단계별로 분석해 주세요:

문제: {문제 설명}

분석 단계:
Step 1: 문제의 핵심 파악
Step 2: 관련 요소 및 영향 분석
Step 3: 가능한 해결책 나열
Step 4: 각 해결책의 장단점 비교
Step 5: 최적 해결책 추천 및 이유"
```

### Few-shot 프롬프트 (예시 기반)
```
"다음 예시를 참고하여 동일한 형식으로 작성해 주세요:

예시 1:
입력: {샘플 입력 1}
출력: {원하는 출력 형식 예시}

예시 2:
입력: {샘플 입력 2}
출력: {원하는 출력 형식 예시}

이제 다음을 작성해 주세요:
입력: {실제 요청}"
```

### 역할 기반 프롬프트 고급 활용
```
"당신은 [역할]입니다.
특성:
- {특성 1}: {구체적 설명}
- {특성 2}: {구체적 설명}
- 제한사항: {하지 말아야 할 것}

이 역할로 다음 질문에 답해 주세요: {질문}"
```

## 구현 가이드

### 프롬프트 최적화 반복 프로세스
1. **초안 작성**: 기본 요청으로 시작
2. **결과 평가**: 무엇이 부족한가?
3. **제약 추가**: 너무 넓으면 좁히기
4. **예시 추가**: 원하는 형식 예시 제공
5. **역할 명확화**: 더 전문적 역할 부여
6. **반복**: 만족할 때까지

### 효과적인 프롬프트 체크리스트
```
□ 명확한 역할이 지정되었는가?
□ 충분한 맥락/배경이 제공되었는가?
□ 요청이 구체적이고 명확한가?
□ 원하는 출력 형식이 지정되었는가?
□ 제한 조건 (분량, 언어, 어조)이 설정되었는가?
□ 필요 시 예시가 포함되었는가?
```

## 고급 패턴

### 프롬프트 라이브러리 구축
```
팀 공용 프롬프트 관리:
- 업무별 검증된 프롬프트 수집
- 버전 관리 (개선 시 기록)
- 팀원 공유 (노션, 구글 문서)
- 결과 품질 평가 기준 포함
```

### 프롬프트 보안 주의사항
```
AI에 입력 금지 항목:
❌ 개인정보 (이름, 주민번호, 연락처)
❌ 영업 비밀, 미공개 사업 정보
❌ 고객사 기밀 정보
❌ 개인 금융 정보

회사 AI 사용 정책 확인 필수
```
