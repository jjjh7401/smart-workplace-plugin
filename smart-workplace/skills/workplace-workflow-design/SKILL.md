# 비즈니스 워크플로우 설계 가이드

## 빠른 참조 (Quick Reference)

### 프로세스 개선 방법론 선택
| 상황 | 방법론 | 특징 |
|------|--------|------|
| 전체 프로세스 혁신 | BPR (업무 재설계) | 완전히 새로 설계 |
| 점진적 개선 | Kaizen (개선) | 지속적 소개선 |
| 오류 최소화 | 6시그마 | 데이터 기반 |
| 빠른 흐름 | 린(Lean) | 낭비 제거 |
| 자동화 | RPA/워크플로우 자동화 | 반복 작업 제거 |

### 프로세스 문제 7대 낭비 (Lean)
1. 과잉 생산
2. 대기 시간
3. 불필요한 이동/운반
4. 과잉 처리
5. 재고 과다
6. 불필요한 동작
7. 결함/오류

## 프로세스 매핑 방법론

### SIPOC 다이어그램
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
SIPOC 다이어그램: {프로세스명}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

S(공급자)    I(입력물)      P(프로세스)       O(출력물)    C(고객)
Supplier     Input         Process           Output      Customer
─────────────────────────────────────────────────────────────
{공급자 1}   {입력 1}   1. {단계 1}       {출력물 1}   {고객 1}
{공급자 2}   {입력 2}   2. {단계 2}       {출력물 2}   {고객 2}
             {입력 3}   3. {단계 3}
                        4. {단계 4}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

예시 (구매 요청 프로세스):
공급자: 현업 부서
입력물: 구매 요청서, 예산 확인서
프로세스: 요청 접수→검토→승인→발주→수령→정산
출력물: 발주서, 납품 확인서
고객: 구매 요청 부서
```

### 플로우차트 표기법
```
[도형 의미]
○  ─── 시작/종료 (타원)
□  ─── 처리/작업 (사각형)
◇  ─── 의사결정 (마름모)
▱  ─── 문서 (문서 모양)
▭  ─── 데이터베이스 (원통)

[흐름 표기]
→  처리 흐름
⋯  점선: 정보 흐름
|  수직선: 책임 경계 (Swimlane)
```

### 스윔레인(Swimlane) 다이어그램 텍스트 표현
```
프로세스: 결재 승인 워크플로우

[담당자 A: 신청자]
○ 시작
↓
□ 지출 결의서 작성
↓
□ 첨부 서류 준비 (영수증, 견적서)
↓
→ [담당자 B에게 전달]

[담당자 B: 팀장]
↓
◇ 금액 기준 검토
  50만 원 이하 → □ 1차 승인 → [담당자 C에게 전달]
  50만 원 초과 → □ 반려 또는 임원 상신

[담당자 C: 재경팀]
↓
□ 예산 확인
↓
◇ 예산 충분?
  YES → □ 최종 승인 → □ 지급 처리 → ○ 종료
  NO  → □ 반려 통보
```

## 업무 프로세스 개선 템플릿

### 현황 분석 (AS-IS)
```
프로세스명: {이름}
분석일: {날짜}
분석자: {이름}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
현재 프로세스 단계
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
단계 | 담당 | 소요 시간 | 문제점
-----|------|----------|-------
{단계 1} | {담당} | {시간} | {문제}
{단계 2} | {담당} | {시간} | {문제}
{단계 3} | {담당} | {시간} | {문제}

총 소요 시간: {시간}
병목 지점: {단계명} ({이유})
오류 발생률: {%}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 개선안 설계 (TO-BE)
```
개선 목표:
- 소요 시간: {현재} → {목표} (X% 단축)
- 오류율: {현재} → {목표}
- 자동화 비율: {목표}%

개선 방안:
① {개선 1}: {방법} → 예상 효과: {시간/비용 절감}
② {개선 2}: {방법} → 예상 효과: {효과}
③ 자동화 대상: {단계} → {도구/방법}

구현 우선순위:
1위: {고효과 + 저비용 항목}
2위: {고효과 + 고비용 항목}
3위: {저효과 항목}
```

## 구현 가이드

### 프로세스 개선 6단계
1. **현황 파악**: 기존 프로세스 문서화 (인터뷰 포함)
2. **문제 도출**: 병목, 낭비, 오류 포인트 식별
3. **원인 분석**: 5Why 기법 또는 피시본 다이어그램
4. **개선안 설계**: 이상적인 TO-BE 설계
5. **파일럿 실행**: 소규모 테스트
6. **전사 적용 및 표준화**

### 5Why 분석 예시
```
문제: 납기 지연 발생

왜 1: 제작 시간이 초과됐다
왜 2: 재료 입고가 늦었다
왜 3: 공급업체 발주가 늦었다
왜 4: 재고 모니터링이 안 돼서 발주 시기를 놓쳤다
왜 5: 재고 관리 프로세스/시스템이 없다

근본 원인: 재고 관리 시스템 부재
해결책: 재고 관리 시스템 구축 + 자동 발주점 알림
```

### 업무 자동화 가능 여부 체크
```
자동화 적합 조건 (3개 이상 해당 시):
□ 규칙이 명확하고 예외가 적다
□ 반복적으로 수행된다 (주 3회 이상)
□ 데이터가 디지털 형태이다
□ 판단보다 처리 위주의 업무이다
□ 수작업 시간이 월 4시간 이상이다

자동화 도구 선택:
- 엑셀 반복 작업: VBA 매크로
- 파일/데이터 처리: Python 스크립트
- 웹 기반 업무: RPA (UiPath, Power Automate)
- 승인/알림: 그룹웨어 워크플로우
```

## 고급 패턴

### 프로세스 KPI 설정
```
프로세스 효율 지표:
- 처리 시간 (Cycle Time)
- 대기 시간 (Wait Time)
- 처리량 (Throughput)
- 품질률 (First Pass Yield)
- 비용/건 (Cost per Transaction)
```

### 변화 관리 (Change Management)
```
프로세스 변경 시 저항 최소화:
① 이해관계자 조기 참여 (설계 단계부터)
② 변경 이유와 기대 효과 명확히 설명
③ 충분한 교육/훈련 시간 제공
④ 파일럿 성과 데이터로 설득
⑤ 챔피언(Change Agent) 발굴 및 활용
```
